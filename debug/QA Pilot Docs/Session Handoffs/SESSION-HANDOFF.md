## Architecture Summary

### Setup File Versions

| Version | Key field | Generated by | Used for |
|---------|-----------|--------------|----------|
| **v1.0** | `student: {...}` (object) | "📥 Generate Setup File" on post-creation prompt | Single student |
| **v2.0** | `students: [...]` (array) | "📥 Generate Class File" toolbar button | Whole cohort or all students |

Both versions are loaded via the same file input on `index.html`. `handleSetupFileLoad()` detects the format by checking for `Array.isArray(data.students)`.

### Class File Format (v2.0)

```json
{
  "setupVersion":    "2.0",
  "generatedDate":   "ISO string",
  "cohort":          "June 2026",   // '' if All Students was selected
  "orgName":         "string",
  "instructorEmail": "string",
  "students": [
    {
      "caseId":      "CASE-00001",
      "name":        "Jane Smith",
      "email":       "jane@example.com",
      "password":    "PBKDF2-salt:hash",
      "role":        "junior",
      "cohort":      "June 2026",
      "dateCreated": "YYYY-MM-DD",
      "active":      true
    }
  ],
  "courseIds": ["qa-onboarding"]
}
```

**NOTE:** `courseIds` is shared across all students in the class file (taken from the first student's enrollments). This is intentional — cohort members are assumed to be in the same courses. Future format extension could add per-student courseIds if needed.

### Student Login Flow (v2.0)

```
Student
───────
1. Opens index.html
2. Clicks "📂 Upload Setup File"
3. Selects the class file (e.g. qa-pilot-class-June-2026-2026-05-26.json)
4. Email-lookup panel appears: "Enter your email address to find your account"
5. Types email → clicks "Find my account →" (or presses Enter)
6. submitClassEmailLookup() finds matching student, builds v1.0-compat payload
7. processSetupFile() creates account, enrolls, auto-logs in → portal.html
```

### Cohort Field in Student Records

- Added `cohort` field to the Add Student modal (text input with datalist autocomplete)
- Stored as `student.cohort` in IndexedDB (empty string if not provided)
- `refreshCohortSuggestions()` reads all students, collects unique cohort tags, populates:
  - `<datalist id="cohort-suggestions">` — autocomplete for the modal input
  - `<select id="class-file-cohort-filter">` — toolbar dropdown for class file generation

```
Admin                                   Student
──────                                  ───────
1. Settings tab → set instructor email
   + org name

2. Add Student modal → fill name,
   email, password, role, cohort → Create

                                        5. Receives email, saves attachment

    Import Data → upload received file

```

## Gotchas for Next Agent

- `generateClassFile()` gets `courseIds` from the **first** student's enrollments. If students have different course enrollments within a cohort, per-student `courseIds` in the v2.0 format would be needed. Not a current issue.
- `_pendingClassData` in `index.html` is a module-level variable (var, not const). It's cleared to `null` after `submitClassEmailLookup()` hands off to `processSetupFile()`.
- `refreshCohortSuggestions()` is async but not awaited when called after student creation — it's fire-and-forget since it's just updating the UI, not critical path.
- The `#class-email-lookup` panel uses inline styles (no CSS class) to keep it self-contained without adding to the stylesheet. Consistent with the existing setup-file-section pattern.
- v1.0 `generateSetupFile(caseId)` still works and is still shown in the post-creation modal prompt. Both v1 and v2 coexist.

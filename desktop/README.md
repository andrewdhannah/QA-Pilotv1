# QA Simulator Desktop

A Windows 11-style desktop simulator built for QA onboarding training. It runs entirely in the browser from a single HTML file — no web server, no install, no frameworks. Drop it in a folder, double-click, and it works.

> Built collaboratively with **GitHub Copilot** and **Claude (Anthropic)**  
> Visual style inspired by [Windows 11 Desktop Simulator](https://github.com/username/windows-11-desktop-simulator) (MIT licence)

---

## What it does

Trainees work inside a realistic-looking desktop environment that simulates the tools they will use on the job:

| App | What it simulates |
|-----|-------------------|
| 📁 **Dynamics** | CRM case investigation — role-based field access, status transitions, validation rules |
| 🧾 **Azure DevOps** | Bug report logging — work item form, required fields, severity/priority |
| ✅ **AC Panel** | Acceptance criteria reference — always-on guide to expected behaviour |
| 🎓 **Training** | Scenario launcher — loads guided QA exercises |
| ⚙️ **Settings** | Theme, background, fidelity mode, and role switcher |

### Desktop features
- Draggable, resizable-by-snap windows (left / right / maximise)
- Win11-style taskbar — centred icons, dot indicators, floating tooltips
- Start menu with pinned apps and role switcher
- Light / dark theme toggle
- Brightness slider
- Notification centre
- Task view overlay
- Lock screen with live clock
- Junior / Senior role system — controls field visibility inside apps
- State persisted to `localStorage` between sessions

## Using the pre-built file

No build step needed. Just open `dist.html` in any modern browser.

```
dist.html   ← everything inlined, fully self-contained
```

Works from `file://`, a USB drive, a network share, or embedded in OneNote.

## Development

### Prerequisites
- [Node.js](https://nodejs.org/) 16 or later

### Project structure

```
Modular OS v4/
├── index.html          # Shell — topbar, taskbar, overlays, desktop icons
├── os.css              # All styles — Win11 tokens, windows, taskbar, menus
├── src/
│   └── os-core.js      # OS engine — state, window management, rendering
├── apps/
│   ├── dynamics.html   # Dynamics CRM mock app
│   ├── ado.html        # Azure DevOps mock app
│   ├── training.html   # Training / scenario launcher
│   └── settings.html   # Settings panel
├── scenarios/          # JSON or JS scenario data files
├── build.js            # Build script — bundles everything into dist.html
├── os.bundle.js        # Dev bundle (generated by build.js, load via index.html)
└── dist.html           # Distribution build — single self-contained file
```

### Build

```bash
node build.js
```

This produces two outputs:
- `os.bundle.js` — for local dev (loaded by `index.html`)
- `dist.html` — the single-file distribution build with all CSS and JS inlined

Open `index.html` in a browser for development (changes to source files are reflected after a rebuild). Open `dist.html` to test the distribution version.

### Why a build step?

Browsers block external resource loading when a page is opened via `file://`. The build script inlines every stylesheet, script, and app HTML so that `dist.html` is completely self-contained and works from any location without a web server.

## Customising scenarios

Scenario files live in `scenarios/`. Each file exports a `SCENARIOS` object keyed by scenario ID. The OS engine exposes `window.QA_OS.loadScenario(id)` inside app iframes so apps can pull in scenario content at runtime.

## Role system

Two roles are available and can be switched from the Start menu or Settings:

| Role | Access |
|------|--------|
| **Junior Investigator** | Read-only on senior fields; cannot resolve or close cases |
| **Senior Investigator** | Full access; must provide Outcome before resolving |

Apps read the current role via `window.parent.QA_OS.getRole()`.

## Theming

Quick Settings (☰ in the taskbar tray) exposes:
- **Light / Dark** theme
- **Brightness** slider
- Wi-Fi / Bluetooth / Focus toggles (decorative)

The Settings app adds background wallpaper variants and a Classic / Win11 fidelity toggle (Classic removes backdrop blur for lower-end machines).

Facilitator Guide

A comprehensive guide for instructors and training coordinators managing QA Pilot Academy students, courses, and assessments.

Platform Overview

QA Pilot Academy is a desktop-based training platform designed for QA onboarding. All student data is stored locally using IndexedDB — nothing is transmitted to external servers. The platform runs entirely from local files, making it deployable without a server infrastructure.

Key Components

  • Portal — The student's landing page showing enrolled courses, available courses, and progress tracking.
  • Course Viewer — The lesson delivery interface with modules, quizzes, and interactive labs.
  • Admin Dashboard — Management interface for student accounts, course configuration, and reporting.
  • Capstone Assessments — Hands-on simulated environments (Dynamics CRM, Azure DevOps) for practical evaluation.

Managing Students

1
Access the Admin Dashboard

Navigate to the Admin Dashboard via the login page link or the "Go to Admin Dashboard" link in the portal sidebar. Use your admin credentials to sign in.

2
Add a New Student

In the Students tab, click "Add Student". Enter the student's name and email address. A unique CASE ID is generated automatically. The student will use their email to log in for the first time.

3
Monitor Progress

The Dashboard displays each student's enrolled courses, completion percentage, quiz scores, and certificate status. Use the course filter to focus on specific programs. Export data as CSV for external reporting.

4
Reset or Deactivate Students

Use the bulk action tools to activate or deactivate students. The "Reset All Students" button (in Settings) clears all data — use with caution. Individual student progress can be reset from the student detail view.

🔒 Demo Account The demo account (demo@qapilot.com) auto-resets on logout. Configure demo course access in Settings → Demo Course Mode. "First-only" shows one course, "Grey-out" shows all but disables enrollment beyond the first.

Course Configuration

Courses are defined in data/content.js with the COURSE_DEFINITIONS array. Each course has modules, sub-modules, and optional quizzes.

Course Prerequisites

Set a prerequisiteId on a course to require completion of another course before enrollment. The portal automatically checks prerequisites and shows helpful messages for locked courses.

Pass Mode & Scoring

Each course has a passMode (assess or audit) and passingScore. In audit mode, students can bypass assessments. Scores are calculated from quiz results and capstone rubric evaluations.

Capstone Assessments

Capstone courses (isCapstone: true) route to capstone-2.html instead of the standard course viewer. They use a simulated OS environment with Dynamics CRM, Azure DevOps, and Outlook apps for realistic testing scenarios.

Data & Privacy

All student data is stored in IndexedDB within the browser. Key data stores include:

  • students — Student profiles (name, email, CASE ID, status)
  • courses — Course definitions seeded from COURSE_DEFINITIONS
  • enrollments — Course enrollment records linking students to courses
  • progress — Per-course progress, completed sub-modules, quiz results, certificates
  • settings — Platform configuration (demo mode, pass scores, etc.)

Data persists in the browser until cleared. Use the "Export Student Data" button in the portal or the CSV export in the admin dashboard to back up records. Resetting or clearing browser data will remove all local information.

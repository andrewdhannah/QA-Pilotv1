Capstone Assessment

This is your final challenge — a hands-on scenario that brings together everything you've learned: testing methodology, acceptance criteria, CRM investigation, and professional bug reporting in Azure DevOps.

📋

What Is the Capstone?

You will step into the role of a QA Tester working on a real-world case investigation. Using the simulated Windows desktop environment, you'll switch between Dynamics 365 (to investigate a case) and Azure DevOps (to file professional bug reports), just like a real QA analyst.
🖥️

The Desktop Environment

When you launch the assessment, a Windows 11-style desktop will load. On the desktop and taskbar you'll find:

🟦 Dynamics 365 — Investigate your assigned case. Switch between Junior/Senior roles to verify role-based permissions. Reference the Acceptance Criteria panel to check expected behaviour.

🐞 Azure DevOps — File bugs with complete repro steps, expected/actual results, and AC references. Track your bugs through the board.

📊 Reports — Review your submission before finalising.

💬 Teams — Receive instructions and context from the team.
🗺️

Your Work Path

Follow these steps in order. Each one builds on the last.
  1. 1
    Open the Brief
    Read the scenario instructions in Teams. Understand the case you've been assigned and what you're testing for.
  2. 2
    Investigate in Dynamics 365
    Open the case. Switch between Junior and Senior roles. Check each field against the Acceptance Criteria. Identify discrepancies.
  3. 3
    File Bugs in Azure DevOps
    For each defect you find, create a professional bug report in ADO. Include numbered repro steps, expected result (with AC reference), and actual result.
  4. 4
    Review & Submit
    Open Reports to review your submitted bugs. When you're satisfied, click Submit for Certification to complete the assessment.
💡
Before you begin: Make sure you've completed all previous modules (Testing 101, Acceptance Criteria, Dynamics 365, Azure DevOps, and Test Planning). The capstone assumes you are familiar with all the tools and concepts.

The desktop simulator will load in full screen. Estimated time: 45 minutes.
